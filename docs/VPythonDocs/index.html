Using VPython to create 3D animations

VPython makes it unusually easy to write programs that generate navigable real-time 3D animations. It is based on the Python programming language which is widely used in introductory programming courses thanks to its clean design, and it is also widely used in science and business.

VPython was originated by David Scherer in 2000. In 2011 David Scherer and Bruce Sherwood initiated the development of GlowScript, a similar programming environment but which runs in a browser. In 2014 it became possible to use RapydScript, a programming language very similar to Python, to support VPython programs in the GlowScript environment. This documentation describes VPython and includes notes on what is not currently available in the GlowScript version.

The version of VPython that runs with a standard installed Python is referred to as "classic" VPython. John Coady has implemented VPython to run in iPython notebooks, with his ivisual module. Aaron Titus has posted a useful summary of the various ways to install and use VPython and here is an update to his ivisual instructions.

Be sure to explore the many GlowScript example programs, written using JavaScript, CoffeeScript, RapydScript, or VPython. The VPython option is based on the RapydScript Python-to-JavaScript compiler. For most programs, RapydScript is nearly the same as Python, but there are differences which can be seen in the RapydScript documentation.

CAUTION

Do not use "wait" as a variable name. All GlowScript languages (VPython, RapydScrpt, CoffeeScript, JavaScript) use "wait" as a special signal for the compilation process. In the VPython case this signal is inserted for you where needed.

For experienced programmers

As a convenience to novice programmers to provide everything that is needed to get started, GlowScript by default imports all of the VPython features and includes standard math functions such as sqrt.

If you want to import selectively, import from the vis or visual module:

from vis import (box, color)
box(color=color.orange)

You can also execute "import vis" or "import visual" but in fact for technical reasons object names not preceded by "vis." or ".visual" are still recognized.

The documentation is written as though "from visual import *" were present.

For those who have used classic VPython

The difference between RapydScript and VPython programs in the GlowScript context is that the VPython option attempts to mimic exactly the syntax and semantics of classic VPython programs, whereas the RapydScript option implements the same semantics for 3D objects as that of JavaScript and CoffeeScript programs. For example, in a RapydScript program the sphere object has a size attribute but no radius attribute, and like box, the default size of the sphere is 1x1x1. Also, changing the length of the axis in a RapydScript program has no effect on the size (except for an arrow, which has a special axis_and_length attribute), but in a VPython program, as in classic VPython, changing the length of the axis also changes the length of the object (the first component of the object's size).

GlowScript by default processes VPython programs as though they had the following statements at the start of the program (which you don't need to include; they will be ignored):

from __future__ import division, print_function
from visual import *
from visual.graph import *

By default, GlowScript treats 3/2 as 1.5, not 1 as in the Python 2.x language, and the print statement must take the form print('hello') rather than the Python 2.x form of print 'hello'.

Many programs written in classic VPython 6 will run in GlowScript without change after being run through a conversion program written in Python. This program converts (x,y,z) => vector(x,y,z) and obj.x => obj.pos.x. These changes are necessary because GlowScript does not recognize (x,y,z) as a vector nor obj.x as a shorthand for obj.pos.x.

Also, in the GlowScript environment pos, size, axis, color, and up are all vectors, and they are somewhat special vectors that track changes in order to trigger correct rendering of the 3D scene when one of these object attributes changes. One result of this architecture is that

         ball.pos += vector(1,2,3)

must be rewritten as

         ball.pos = ball.pos + vector(1,2,3)

The conversion program makes these changes, and similar changes for -=, *=, and /=. The program also converts scene.mouse.getclick() => scene.waitfor('click'), which works in both environments.

In GlowScript you can use the shorthand "vec" for "vector", and it is intended to have classic VPython also allow either form. If you wish to use a GlowScript program containing "vec" in the classic VPython environment, just add the statement "vec = vector" at the start of the program.

In addition to classic VPython and this new VPython in the GlowScript browser environment, a third way to run a VPython program is in an IPython notebook, in a browser, thanks to the ivisual module created by John Coady.

Limitations

A few VPython objects are not currently available in GlowScript: convex, faces, frame, and the 3D text object. The GlowScript objects vertex, triangle, and quad represent a more powerful alternative to faces and are candidates for implementing in classic VPython. Many applications of frame can be handled with the GlowScript compound object. The extrusion object (and the paths and shapes libraries) currently has many fewer features than the classic version.

One way to deal with differences is to check the elements of the "version" variable that is available in both classic VPython and the GlowScript version and do different things in the different environments:

Classic VPython:    version is ['X.Y', 'release']
GlowScript VPython: version is ['X.Y', 'glowscript']

The curve and points objects are slightly different in the GlowScript environment, but with a little care it is possible to use these objects in such a way that they will also work in classic VPython. Note that in GlowScript the list of points in a curve object is not a numpy array, so that a loop is required to change all of the points. However, GlowScript programs compile to JavaScript, whose loops are much faster than classic Python loops.

To handle mouse events one cannot use scene.getevent() but must use scene.bind(), which is available both in VPython 6 and in GlowScript. Also available are scene.pause() and scene.waitfor('click') and related options.

It is possible in the GlowScript environment to place widgets such as buttons or sliders on the web browser page, but the mechanism is that of HTML and jQuery, not the wxPython component of classic VPython. Currently there isn't an easy way to display a button or slider in a way that will work both in the classic VPython environment and in the GlowScript environment.

It is not possible to import arbitrary Python modules such as numpy, and any VPython program that uses numpy will have to be modifed. However, loops are fast in the JavaScript language to which GlowScript programs compile, so if you are using numpy solely for the speed of array manipulation, you may be able to replace a numpy calculation easily and efficiently with a loop.

An existing self-contained Python module can be processed in GlowScript to its JavaScript equivalent and then placed on a server to be used as a library for GlowScript programs. (When editing a GlowScript program you can choose "Share this program" to output the JavaScript code.)

Text output

The print statement is similar to the Python print statement, but see the documentation on additional text output capabilities available in the web browser environment.

Additional capabilities

This VPython Help includes comments on additional capabilities offered by GlowScript that are not currently available in classic VPython. You can use these features with the understanding that the resulting GlowScript program may not run in the classic environment without modification. Examples include a more powerful form of faces (using vertex, triangle, and quad objects), compound objects that can be displayed very efficiently, and the ability to clone an object. The attach_trail and attach_arrow options are more powerful subsitutes for the classic VPython make_trail (which is also supported); attach_trail and attach_arrow are candidates for inclusion in classic VPython.

You can set the size attribute (or length, width, and height) of any object arbitrarily, so for example you can make a cylinder or cone object have an elliptical cross section. Although GlowScript supports the classic VPython ellipsoid object, you can also make an ellipsoid simply by setting the size (or length, width, and height) of a sphere object appropriately. And of course you can take advantage of the HTML/jQuery capabilities of the GlowScript browser environment to create rich web pages, and optionally export a VPython program to your own web site.

Note too that color, like pos and size and axis and up in classic VPython, is a vector in GlowScript, so that red, green, and blue are the x, y, and z components of color. This is a candidate for importing into classic VPython.
